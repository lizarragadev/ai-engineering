# Script: Arquitectura de LangChain — Componentes y Filosofía de Diseño

1. Introducir LangChain como un framework de orquestación diseñado específicamente para flujos de trabajo LLM.
2. Mostrar un diagrama de arquitectura de alto nivel etiquetando LLMs, Prompts, Chains, Indexes y Agents.
3. Definir cada componente de manera sucinta:
   - **LLMs:** Interfaces a modelos de lenguaje.
   - **Prompts:** Plantillas y gestión de prompts de entrada.
   - **Chains:** Encapsulan lógica multi-paso y orquestan el flujo de componentes.
   - **Indexes:** Facilitan la recuperación y búsqueda de documentos.
   - **Agents:** Habilitan la toma de decisiones dinámica e integración de herramientas.
4. Usar analogías (ej., chains como flujos de trabajo en pipeline) para clarificar conceptos.
5. Presentar el flujo de interacción de componentes dentro de un pipeline RAG típico.
6. Resumir la conclusión clave: entender estas abstracciones es vital para sistemas de IA modulares.

**Puntos de Verificación:**
- Si puedes explicar brevemente el rol de cada componente de LangChain, estás preparado para el trabajo práctico.
- Si el diagrama de arquitectura modular tiene sentido lógico, estás siguiendo bien.
